\# University Database Audit, Normalization and BI Report



\## **Overview**

This project audits, corrects, and enhances a university management database. Inadequate normalization, inconsistent data types, lack of foreign key associations, and missing primary keys, foreign keys plagued the original database.



The project applies structural fixes and creates analytical SQL reports to support academic, administrative, and operational decision-making.



\## **Key Improvements**

* Added primary keys to all tables
* Established 28 foreign key relationships
* Fixed incorrect data types 
* Removed redundant and duplicate tables
* Improved normalization and ERD design



\## **Analytical Reports**

\- Top students by CGPA per department

\- Faculty workload and research productivity

\- Course performance and grading trends

\- Library usage and book popularity

\- Room and building utilization analysis

\- Attendance pattern analysis



\## **Tools \& Technologies**

\- SQL Server (T-SQL)

\- Relational Database Design

\- Normalization (3NF)

\- Business Intelligence Reporting



\## **How to Use**

1\. Execute schema files in the `schema/` folder

2\. Apply migration scripts in the `migrations/` folder

3\. Run analytical queries from the `reports/` folder
